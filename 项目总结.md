# 知识图谱提取与可视化系统 · 项目总结

> 上传 PDF / Word 文档，本地大模型自动提取实体与多元关系，并以 **2D 交互式力导向网状图** 可视化（加载动画、悬停看详情卡片、单击高亮邻居与关联边、图例聚光灯、弹性拖拽、刷新不丢、一键导出离线 HTML）。整套链路（前端 + 后端 + 大模型推理）全部跑在本地，不依赖任何云端 API。

## 二、模型选型与优点 ★

本项目选用 **`Gemma4-12B-QAT-Uncensored-HauhauCS-Balanced`**（Q4_K_M GGUF），搭配 **`mtp-gemma-4-12B-it`** 投机解码草稿头。它特别适合"本地知识图谱提取"这个场景，原因如下：

### 1. QAT 4-bit 量化 —— 小体积、近全精度质量

Gemma 4 是**量化感知训练（QAT）**的，专门为 ~4-bit 优化。Q4_K_M 是作者认定的"甜点"：6.9GB 即可装下 12B 模型，**质量接近全精度**，更高精度的量化只会徒增体积而无实质收益。这意味着一台普通消费级 GPU（甚至 8–12GB 显存）就能流畅本地推理。

### 2. Balanced 变体 —— 先推理、再作答

作者推荐的 **Balanced** 变体专为 agentic coding、reasoning、可靠任务调校，特点是 **"reasons before answering"（先思考再作答）**。对知识图谱提取这种需要"理解语义 → 判断实体 → 挖掘多元关系"的任务，推理能力直接决定图谱质量（关系是否丰富、是否星型退化）。模型稳定、听从指令，输出格式可控。

> **代价**：reasoning 会先产生一段思考链，**思考链和最终 JSON 共享 `max_tokens` 预算**（本项目据此把 `max_tokens` 设到 12288，见后文踩坑）。

### 3. MTP 投机解码 —— 提速约 60%，质量不变

自带 **MTP（Multi-Token-Prediction）草稿头**做投机解码：草稿头一次预测多个 token，主模型验证，**速度提升约 60% 且输出完全一致**（每个 token 都经验证，纯提速不降质）。知识图谱提取是分块多次调用，提速收益叠加明显。

### 4. 256K 超长上下文

12B dense，原生 **256K（262144）上下文**，理论上可处理极长文档。本项目当前为控制耗时把单文档分块上限设为 6 块，但模型本身的天花板很高，后续扩展长文档处理无压力。

### 5. Uncensored —— 0/465 拒绝

经自动 + 人工拒绝率测试 **0/465**，不因内容敏感而拒答。提取任务（尤其历史、政治、传记类文档）不会因"安全过滤"丢失实体或关系。

### 6. 作者端到端调校的采样参数 —— 本项目的关键

作者强调这套参数"是模型的一部分，不是 Gemma 默认值"，专门为该 build 调校：

| 参数                 | 值      | 在本项目的作用                                                      |
| -------------------- | ------- | ------------------------------------------------------------------- |
| `temperature`        | 0.6     | 关系挖掘的随机性与稳定性平衡                                        |
| `top_k`              | 64      | 作者推荐                                                            |
| `top_p`              | 0.9     | 作者推荐                                                            |
| `min_p`              | 0.05    | 作者推荐                                                            |
| **`repeat_penalty`** | **1.1** | **防止 reasoning 陷入 "Wait…Wait…" 死循环的关键**（本项目实测验证） |

> 这套参数是项目调试中反复踩坑后才回归的"正解"。早期用 stock 默认值时，模型频繁出现思考死循环 / 输出截断，套上作者参数（尤其 `repeat_penalty 1.1`）后稳定。

### 7. 生态兼容

兼容 llama.cpp、LM Studio、Jan、koboldcpp 等 GGUF 运行时。本项目用 llama.cpp 的 `llama-server`，OpenAI 兼容接口对接零成本。

## 五、核心功能与数据流

后端 `main.py` 暴露 3 个接口：

| 接口                      | 作用                                                                                                |
| ------------------------- | --------------------------------------------------------------------------------------------------- |
| `POST /api/upload`        | 接收 PDF/DOCX，解析为 Markdown（PDF 按页 `## 第 N 页`，Word 保留标题层级），存一份到 `uploads/*.md` |
| `POST /api/extract-graph` | 文本分块 → 每块独立调 LLM 抽取图谱 → 按实体名合并去重 → 结构质量统计 → 返回 `{nodes, links}`        |
| `GET /api/health`         | 健康检查                                                                                            |

**完整用户流程**：

1. **上传**：拖拽上传 → 后端提取文本 → 回显前 800 字预览。
2. **提取**：点「提取知识图谱」→ 触发**等待动画**（SVG 节点网络脉冲 + 进度条 + 分阶段文字轮播）→ 后端分块抽取（短文档单次，长文档分块合并）。
3. **可视化**：`force-graph` 渲染 2D 网状图：
    - 节点按 9 类配色（人物/组织/概念/事件/地点/技术/作品/领域/其他），带发光与白色描边常显标签；
    - **悬停节点**：左下角浮出详情卡片（色点/名称/类别/描述），鼠标移开自动消失；
    - **单击节点**：高亮该节点 + 其邻居节点 + 关联边（亮青色加粗），其余淡化；再点同一节点取消；
    - **图例聚光灯**：点击某类 → 该类放大+强发光+白描边、其余隐入背景，连线高亮加粗（同时自动取消节点高亮，避免冲突）；
    - **弹性拖拽**：长按拖动节点，相连节点像弹簧一样被带着动，松手后整图弹性回弹；
    - 滚轮缩放、拖背景平移、布局稳定后自动适配视野；
    - **刷新不丢**：文档与图谱存入浏览器 localStorage，刷新页面无需重传重提。
4. **导出**：点「下载图谱 (HTML)」→ 生成**完全自包含的离线 HTML**（内嵌 force-graph 全量包 + 图谱数据 + 渲染逻辑），双击即可在浏览器离线打开，交互与系统内一致。

**实体类别（category）**：人物、组织、概念、事件、地点、技术、作品、领域、其他。

## 六、关键技术实现与踩坑（项目精华）

### 后端 · 反星型 + 分块 + 容错

- **反星型 + 连通性 system prompt**：硬性结构约束（**连通性最高优先**——整图必须单一连通、严禁离群点/多子图；任何节点连接数 ≤ 总关系数 40%；每实体 ≥ 2 连接；平均度 ≥ 2）+ 7 个关系挖掘维度（归属/位置/协作/因果/时序/相似/产出）+ few-shot 网状示例。从源头避免"星型退化"与"离群点"。
- **连通性后端兜底**：prompt 是软约束，模型仍可能漏。合并后用 `_largest_component` 做 BFS 连通分量分析，**只保留最大连通分量**，丢弃离群节点与互不相连的小子图，保证返回前端的图一定连通（日志打印丢弃数量）。
- **分块提取 + 合并去重**：`SINGLE_CALL_MAX=3000` 单次阈值；超长则 `CHUNK_SIZE=2000` / `OVERLAP=200` / `MAX_CHUNKS=6` 切块，每块独立抽取，再按实体名归并、重映射关系、丢弃自环与未知引用。
- **容错 JSON 提取**：模型是 reasoning 型，可能先吐思考链再输出 JSON 或包代码块。`_parse_graph_json` 先试整体解析，失败则用**括号匹配**抓第一个平衡的 `{...}`，抗截断、抗前后文污染。
- **重试 + 采样**：单块失败时提高 `temperature`（0.6→0.8）重试一次跳出循环；全参数按作者推荐（`repeat_penalty 1.1` 防 death-loop）。
- **结构质量日志**：合并后打印平均度数、最大度数占比、孤立节点数，给出 `✓ 网状` / `⚠ 偏星型` 判定，便于排查。

### 后端 · 关键参数调优历程

输出被截断问题层层递进，根因一直在变，最终定位到 **reasoning 共享预算**：

| 现象                                            | 根因                                    | 修复                                 |
| ----------------------------------------------- | --------------------------------------- | ------------------------------------ |
| prompt 都装不下                                 | ctx 太小                                | ctx 8192 → **16384**                 |
| JSON 写到一半被砍                               | `max_tokens` < 完整 JSON                | 提升 `max_tokens`                    |
| 思考链重复 "Wait…" 死循环                       | 无重复惩罚                              | **`repeat_penalty 1.1`**（作者推荐） |
| 思考链吃掉输出预算，JSON 刚开头就 `length` 截断 | **reasoning 与 JSON 共享 `max_tokens`** | `max_tokens` → **12288**             |

### 前端 · force-graph 踩坑全集

| 坑              | 现象                                    | 解决                                                                                                                  |
| --------------- | --------------------------------------- | --------------------------------------------------------------------------------------------------------------------- |
| 构造方式        | `ForceGraph(el)` 不创建 canvas          | kapsule 两阶段：`ForceGraph()(el)`                                                                                    |
| 模拟冻结        | 切换图例后画面不变                      | 力模拟冷却后 canvas 不重绘，切换时调 `d3ReheatSimulation()` 触发重绘                                                  |
| 视图跳变        | reheat 后 `zoomToFit` 重置缩放          | `onEngineStop` 加 `initialZoomDone` 标志，仅首次布局适配视野                                                          |
| 按钮被盖        | 右上角下载按钮看不见                    | force-graph 动态 `appendChild` 的 canvas 盖住了前面的元素；下载主入口改放侧边栏（纯 HTML 区），toolbar 提高 `z-index` |
| Vue patch 冲突  | 悬停卡片高频切换，刷一屏 `insertBefore` 崩溃 | 把所有浮层（卡片/工具栏/加载）移出 `graph-container`，用一个 `graph-wrap` 包裹；canvas 容器内 0 个 Vue 节点，彻底隔离 |
| onBackgroundClick 吞单击 | 单击节点无反应（被判成拖拽）   | 注册 `onBackgroundClick` 会启用 force-graph 拖拽检测，pointer 微动即判 drag、pointerup 直接 return；移除它，取消改"再点节点 toggle" |
| 边高亮引用不匹配 | 单击节点邻居亮了、边却不亮             | force-graph 内部对 link 端点重新包装，对象引用对不上；改用 `link.source.id === pinnedNode.id` 的 **id 字符串比较**    |
| reheat 漂移      | 每次点击高亮，节点越漂越远出画面       | `onEngineStop` 后给所有节点设 `fx/fy` 锁定位置；reheat 只触发重绘、不移动节点                                        |
| non-finite 崩溃 | `createRadialGradient`/`arc` 报非有限值 | `nodeCanvasObject` 开头校验 `globalScale`/坐标非有限则跳过                                                            |
| 循环引用        | 导出时 `JSON.stringify` 报 circular     | force-graph 原地给节点加 `neighbors/links/x/y`；导出前**清洗**重建干净 `{nodes, links}`                               |

### 前端 · 离线 HTML 导出

- force-graph 的 `package.json` 用 `exports` 封了子路径，`?raw` import 失败 → 把 UMD 包复制到 `public/`，运行时 `fetch('/force-graph.min.js')` 取文本内联（`_libCache` 缓存）。
- 模板里的 `</script>` 会让 Vue SFC 解析器误判 `<script setup>` 结束 → 写成 `<\/script>` 绕过。
- 导出的 HTML 自包含：force-graph 全量包 + 清洗后的数据 + 配色/绘制/悬停高亮逻辑 + 图例 + 统计，**离线双击即用**。

### 前端 · 弹性拖拽与持久化

- **弹性拖拽（三段式锁定）**：默认锁住所有节点 `fx/fy`（防点击 reheat 漂移）；`onNodeDrag` 时只释放"非被拖节点"的锁定，让 link force 像弹簧拉着它们跟随被拖节点；`onNodeDragEnd` 释放被拖节点让整图弹性回弹；回弹稳定后 `onEngineStop` 重新锁住。既保住"拽一个、带动一片、松手回弹"的手感，又不丢"点击高亮不漂移"。
- **刷新持久化**：文档文本、字符数、文件名、清洗后的图谱数据存入浏览器 `localStorage`（键 `kg_state_v1`），`onMounted` 读回并重新渲染，刷新页面无需重传重提。
- **交互互斥**：点击图例自动清空节点单击的高亮（`toggleCategory` 清 `pinnedNode`），避免两类高亮叠加。

## 八、可扩展方向

- **真实提取进度**：当前加载动画是阶段轮播（前端拿不到分块中间进度）。可用 SSE / WebSocket 把后端每块进度推给前端，做成真实百分比进度条。
- **更长文档**：调大 `MAX_CHUNKS` 或利用模型 256K 上下文做整文档单次抽取（权衡耗时）。
- **图谱编辑**：前端增加节点/关系的增删改，支持人工校正后再导出。
- **多格式导出**：在 HTML 之外增加 PNG 图片、JSON 数据、GraphML 等导出。
- **多模型切换**：`model/gemma-4-yy/` 下另有 31B 模型，可加切换入口对比图谱质量。
